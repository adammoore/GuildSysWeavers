The Guild of Systems Weavers is governed as lightly as possible, with structure added only as clearly needed.

## Current governance

The guild is in its founding phase. Current governance is:

- **Stewardship** is provided by the founding maintainers
- **Contributions** are accepted through GitHub pull requests, reviewed by maintainers
- **Decisions** about scope, structure, or direction are discussed in issues before implementation
- **All governance discussions** are conducted in the open, in the repository

## Decision-making

See [decision-making model](decisions) for how decisions are made.

## Principles

Governance in the guild is guided by:

- **Minimum viable structure** — add governance only when its absence is causing problems
- **Transparency** — decisions and their rationale are documented publicly
- **Contributor voice** — people who contribute have standing to shape the direction
- **Reversibility** — prefer decisions that can be revisited over those that cannot

## Contributing to governance

If you have a suggestion for how the guild should be governed, [open an issue](https://github.com/adamvialsmoore/GuildSysWeavers/issues/new?template=governance-suggestion.yml).
